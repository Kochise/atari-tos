2. Worum geht's?
================
Besitzern von Atari-Rechnern (das sind diese popeligen grauen oder weiŞen
Kästen mit den vielen Knöpfen und den lustigen Bomben auf dem Bildschirm),
die als Betriebssystem TOS2.06 oder 3.06 fahren, werden beim Kaltstart
ihres Computers mit einem Signet auf dem Bildschirm begrüŞt.
Im Regelfall wird dies das Fuji-Logo von Atari sein, aufgeklärtere
Zeitgenossen haben dieses bereits durch den Flower-Power-Slogan der
90er Jahre (der aber auch schon vor 10 Jahren seine Berechtigung gehabt
hätte) "Gib DOS keine Chance" ersetzt.

Immer nur Atari-Logo ist aber langweilig, DOS hat eigentlich sowieso
keine Chance mehr und andere Bildchen konnte man bisher nicht verwenden.
Kurz und knapp: jetzt kann man. TOS-ICON ist nämlich genau dafür und nur
dafür zu gebrauchen. Wahnsinn, gell?

4. Kurzbeschreibung
===================
Der Editierteil dürfte klar sein (linke/rechte Maustaste), ebenso das
Scrollen, Löschen und Füllen. Allenfalls die 6 Buttons am rechten Rand
bedürfen vielleicht einer Erklärung.

4.1. TOS -> Icon
Liest aus einer TOS-Datei, die in 256K- oder 512K-Form vorliegen muŞ, das
darin gespeicherte Icon direkt in den Editor.
Falls auf dem Schirm zu diesem Zeitpunkt nur Schrott erscheint, stimmt
irgendetwas mit der Datei nicht (falsches Format, neuere oder ausländische
TOS-Version). Die folgende Funktion darf dann AUF KEINEN FALL auf diese
Datei angewendet werden!

4.2. Icon -> TOS
Geht den umgekehrten Weg, d.h. manipuliert direkt die anzugebende TOS-Datei,
indem es darin das Icon durch das momentan im Editor befindliche ersetzt.
Da Atari mittlerweile beim Booten eine Prüfsumme über das ROM bildet
(ausdrückliches Lob von meiner Seite), muŞ diese nach der Manipulation
angepaŞt werden. Dafür ruft TOS_ICON das Programm TOSCRC auf, das im
gleichen Verzeichnis stehen muŞ wie TOS_ICON.

4.3. PIC/PAC laden
Eine Datei im Bildschirm- oder STad-Format wird eingeladen.

4.4. Icon ausschneiden
Schneidet aus der geladenen Bilddatei ein Icon aus.

4.5. Icon speichern
Das momentan im Editor befindliche Icon wird auf Diskette/Festplatte
gespeichert. Dazu wird eine zu überhaupt nichts kompatible Form verwendet,
die nur aus den reinen Pixelinformationen besteht. Die Dateilänge ist
konstant 1032 Bytes und die Default-Dateiendung ".IKN".
