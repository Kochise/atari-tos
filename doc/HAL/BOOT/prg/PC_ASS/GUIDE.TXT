
Programmbeschreibung:

Der PC_ASS ist eine Kombination aus einem Assembler und einem 
Disassembler für die Maschinensprache der Sharp Pocket Computer. 
Er dient dazu, die lästige Zettelwirtschaft, die bei der Entwick-
lung der Maschinenprogramme entsteht, zu verhindern. AuŞerdem 
können die Programme auf Diskette gespeichert werden, so daŞ sie 
dort sicher untergebracht und jederzeit abrufbereit sind. Auch 
das Ausdrucken der erstelen Listings, sowie das Löschen, das 
Korrigieren und das Einfügen von Programmzeilen ist möglich.

Der PC_ASS ist vollständig in GFA-Basic geschrieben, so daŞ es 
jedem möglich ist, individuelle Veränderungen vorzunehmen. Für 
diejenigen, die keinen GFA-Basic Interpreter besitzen ist, eine 
compilierte Version auf der Diskette enthalten. Ich wünsche allen 
Benutzern von PC_ASS frohes Schaffen und All Reset freie Pro-
gramme.

Der Autor:  Stefan Schwardt



Der Disassembler:

Hier können Sie bereits bestehende oder neue Programme im HEX-
Code eingeben. Das Programm versucht dann sofort, diese zu 
disassemblieren. Die Hex-Codes können über die rechte Zehner-
tastatur eingegeben werden. Diese habe ich jedoch in eine Hex-
Tastatur umgewandelt, um das eingeben der Codes zu erleichtern. 
Durch die neue Belegung ist es nicht mehr nötig, die in der Hex-
Form vorkommenden Buchstaben (a-f) über das normale Buchstaben-
feld einzugeben. Die neue Belegung ist wie folgt:

"("="a", ")"="b", "/"="c","*"="d", "-"="e", "+"="f"

Bei der Eingabe bedarf es keiner Bestätiung durch Return, da das 
Programm selbstständig das Zeilenende erkennt. Auch andere Tasten 
wie Space brauchen nicht betätigt zu werden. So ist ein flüssiges 
Eingeben der Codes möglich. Allerdings müŞen die Hex-Codes aus 
zwei Zeichen bestehen. Sie müssen also zum Beispiel anstelle von 
"2" "02" eingeben.

Sollte sich beim Eingeben ein Fehler eingeschlichen haben, so 
kann dieser durch betätigen der DELET-Taste korrigiert werden.
Doch Achtung: DELET löscht jeweils die ganze zu letz eingegebene 
Zeile. Ist das Programm vollständig eingegeben, so kann der 
Disassembler über die ESC-Taste verlassen werden. Das Programm 
befindet sich nun im Speicher und kann über die Option SPEICHERN 
auf Diskette abgelegt werden.

Vorsicht: Es können nicht mehr als 8000 Zeilen im Speicher 
          aufgenommen werden (es sei denn, Sie ändern die 
          Dimensionierung von Byte$(8000,3) und Buff2%(8000,3)). 
          Es findet keine Überlaufsabfrage statt.


Der Assembler:

Der Assembler ermöglicht das Eingeben von Programmtexten in 
Assemblerformat. Dabei muŞ jedoch der Syntax der Befehle peinlich 
genau befolgt werden (siehe Befehlssatz anzeigen). Die Eingabe 
einer Zeile muŞ hier, anders als beim Disassembler, mit RETURN 
abgeschloŞen werden. Das Programm gibt zur Bestätigung die Zeile 
dann in ihrem entgültigen Format aus. Die Korrektur einer bereits 
mit Return bestätigten Zeile ist im Assembler nicht möglich 
(siehe Zeilen korrigieren). Jedoch können Sie innerhalb einer Zeile 
mit DELET Korekturen vornehmen. Verlassen können Sie den 
Assembler indem Sie statt einem Befehl nur ein "x" eingeben und 
mit RETURN bestätigen. Nach Verlassen des Assemblers ist das 
Programm wie beim Disassembler im Speicher abgelegt und kannt mit 
SPEICHERN auf Diskette geschrieben werden.

Vorsicht: Auch hier gilt nicht mehr als 8000 Zeichen und keine 
          Überlaufsabfrage.


Programm zeigen:

Bei Anwahl dieser Option wird ein im speicherbefindliches Pro-
gramm Zeile für Zeile auf dem Bildschirm ausgegeben. Die Ausgabe 
kann durch betätigen der SPACE-Taste unterbrochen werden. Wird 
die SPACE-Taste erneut betätigt, so wird die Ausgabe fortgesetzt.


Programm drucken:

Wenn Sie diese Option anwählen, dann wird ein ein, zwei oder drei 
spaltiges Programmlisting vom Drucker ausgegeben. Jedoch ist das 
Programm auf einen Panasonic KX-P 1091 abgestimmt. Eine Drucker-
anpassung ist allerdings möglich. Dazu muŞ die Zeile, die in der 
Procedur DRUCKEN mit " 'Tabulator' " gekennzeichnet ist, an den 
jeweiligen Drucker angepaŞt werden. Sie stellt die Tabulatorweite 
des Druckers ein. AuŞerdem muŞ die Zeile, die mit " 'Tabulator 2' 
" gekennzeichnet ist, angepaŞt werden. Hier wird mir chr$(9) ein 
Tabulatorsprung ausgeführt.


Speichern:

Es kann ein im Speicher befindliches Programm abgespeichert wer-
den.


Laden:

Bereits abgespeicherte Programme können zur weiteren Bearbeitung 
wieder in den Speicher geladen werden. Dabei wird jedoch ein 
Programm, daŞ sich schon vorher im Speicher befunden hat, 
gelöscht.


Zeilen einfügen:

Mit dieser Option können neue Zeilen in das bereits bestehende 
Programm eingefügt werden. Sie werden zuerst nach der Zeile 
gefragt, hinter der die neuen Zeilen eingefügt werden sollen. 
Dann können Sie die neuen Zeilen eingeben, allerdings nur im Hex-
Code, da diese Procedur den Disassembler benutzt. Beendet wird 
die Eingabe wie im Disassembler durch die ESC-Taste.

Vorsicht: Es können nicht mehr als 8000 Zeilen im Speicher 
          aufgenommen werden. Es erfolgt auch hier keine 
          überlaufsabfrage.


Zeilen löschen:

Hier können Programmzeilen gelöscht werden. Sie werden nach der 
Start- und der Endzeile gefragt.

Vorsicht: Die Zeilen sind für immer verloren.


Zeilen korrigieren:

Wenn sie nach Fertigstellung eines Programms noch Fehler ent-
decken, so können Sie mit dieser Procedur einzelne Zeilen 
korrigieren. Nach Eingabe der zur korrigierenden Zeilen wird die 
alte Zeile angezeigt. Sie können nun die neue Zeile eingeben 
(auch hier wieder im Hex-Code). Die alte Zeile wird nun durch die 
neue ersetzt.


Befehlssatz anzeigen:

Diese Option gibt den Befehlssatz in tabellenform auf dem 
Bildschirm aus. Der Syntax der einzelnen Befehle ist für den 
Assembler sehr wichtig. Deshalb sollten Sie sich einen Ausdruck 
des Befehlssatzes erstellen, denn diese Funktion kann vom 
Assembler aus nicht aufgerufen werden.


Befehssatz drucken:

Befehlssatz drucken druckt den Befehlssatz in Tabellenform aus. 
Es ist keine Druckeranpassung nötig.


Beispielprogram:

Das Beispielprogram ist der Maschienenteil des nachfolgenden 
Programs. Es erzeugt einige Töne und ist für dem Sharp PC-1450.

5 DIM B$(0)*30
10 POKE 23569,&3,&FD,&2,&FE,&67,&FF,&38,&E,&12,&5F,&61,&10,&DF, 
             &4E,&FF,&60,&E9
20 POKE 23586,&DF,&42,&2D,&10,&DA,&67,&FF,&38,&5,&42,&DA,&2D,&1B,&37
30 FOR I=50 TO 100 STEP 2
40 POKE 23570,253,2,22:POKE 23583,I:CALL 23569
50 NEXT I
