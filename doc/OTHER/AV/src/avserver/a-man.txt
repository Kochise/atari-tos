1. A-MAN ist ein Programm, das unter allen TOS-Systemen neue 
AES-Funktionen zur Verfügung stellt. A-MAN ist Freeware und darf 
eigenen Programmen ohne weiteres beigelegt werden. A-MAN existiert in 
einer speziellen Version, die au×er dem eigenen Cookie ('AmAN') noch 
einen weiteren Cookie ('WICO') anlegt. Diese Version liegt dem 
WinCom-Archiv bei. Die Funktionalität der im jeweils aktuellen 
WinCom-Archiv enthaltenen A-MAN-Version stimmt mit der jeweils 
aktuellen A-MAN 'standalone'-Version überein.

2. A-MAN
Die Existenz von A-MAN erfährt man durch Suche nach dem Cookie 'AmAN',
dessen Wert ein Zeiger auf folgende Struktur ist:

typedef struct
{
 long date;

 short *dcolor_a;
 short *dcolor_b;

 short *currxywh;
 short *kind;
 short *owner;
 char **name;
 char **info;

 signed char *menu_id;
 OBJECT **menu_tree;
 short  *flags;
}A_MAN;

'date' ist BCD kodiert und stellt Versionsnummer und -Datum wie folgt
dar:
 aabbccdd
 aa-Versionsnummer a.a
 bb-Tag
 cc-Monat
 dd-Jahr
 14030195 bedeutet also Version 1.4 vom 3.1.1995

Ist die Versionsnummer mindestens 1.4, so stehen folgende Funktionen
zur Verfügung:
(Die Versionsnummer 1.4 mu× nicht explizit geprüft werden, da ohnehin
 erst ab dieser Version der Cookie 'AmAN' angelegt wird)

(Für die im folgenden erwähnten wind_get()-Funktionen gilt: Die
 AES.LIB von PureC liefert bei den meisten Funktionen fehlerhafte
 Werte! Verwenden sie stattdessen eine eigene wind_get()-Funktion.
 Wenn ihr PureC-Programm "#include <aes.h>" enthält, so könne sie
 wind_get auch durch die untenstehende wnd_get Routine ersetzen.
 Diese Routine berücksichtigt alle Rückgabewerte.)

ACHTUNG! Für die Fenster des im TOS integrierten Desktop sind 
keinerlei Informationen über A-MAN verfügbar!

'dcolor_a' ist ein über die Fensterelementnummer indiziertes Feld und
gibt die Farbe dieses Elements für ein aktives Fenster an.
Diese Werte können auch über wind_get(WF_DCOLOR) erfragt werden. Die
entsprechende AES-Funktion wird von A-MAN zur Verfügung gestellt.

'dcolor_b' wie dcolor_a für inaktive Fenster
Diese Werte können auch über wind_get(WF_DCOLOR) erfragt werden. Die
entsprechende AES-Funktion wird von A-MAN zur Verfügung gestellt.

'currxywh' ist ein über 4*Fensterhandle indiziertes Feld. Jeweils vier
aufeinanderfolgende Einträge geben die momentanen Ausma×e des Fensters
an. Einträge für nicht vorhandene Fenster sind undefiniert!
Will man also die Ausma×e des Fensters mit Handle 'wid' auslesen,
lauten die Abfragen:
 x=currxywh[4*wid];
 y=currxywh[4*wid+1];
 w=currxywh[4*wid+2];
 h=currxywh[4*wid+3];
Der Vorteil gegenüber der herkömmlichen wind_get-Abfrage ist der, da×
dieses Feld auch problemlos in Interrupts etc. ausgelesen werden kann.

'kind' ist ein über das Fensterhandle indiziertes Feld. Es gibt die
bei 'wind_create' angemeldeten Fensterelemente wieder. Einträge für
nicht vorhandene Fenster sind nicht definiert!
Diese Werte können auch über wind_get(wid, WF_KIND, &kind...) erfragt
werden. Die entsprechende AES-Funktion wird von A-MAN zur Verfügung
gestellt.

'owner' ist ein über das Fensterhandle indiziertes Feld. Es gibt den
Eigner des Fensters an. Einträge für nicht vorhandene Fenster sind
nicht definiert!
Die owner-Werte werden von A-MAN NICHT über wind_get zur Verfügung
gestellt!

'name' ist ein über das Fensterhandle indiziertes Feld. Es gibt die
Zeiger auf die NAME-Texte der Fenster an. Einträge für nicht
vorhandene Fenster sind nicht definiert!
Die Zeiger können auch über wind_get(wid, WF_NAME, &hi, &lo..) erfragt
werden. Die entsprechende AES-Funktion wird von A-MAN zur Verfügung
gestellt.

'info' ist ein über das Fensterhandle indiziertes Feld. Es gibt die
Zeiger auf die INFO-Texte der Fenster an. Einträge für nicht
vorhandene Fenster sind nicht definiert!
Die Zeiger können auch über wind_get(wid, WF_INFO, &hi, &lo..) erfragt
werden. Die entsprechende AES-Funktion wird von A-MAN zur Verfügung
gestellt.

'menu_id' ist ein über die AES-ID indiziertes Feld und gibt die
menu_id eines installierten Accessories an. Für AES-IDs, die keinem
installierten Accessory zuzuordnen sind, wird -1 geliefert. Die Werte
können auch über menu_register(-1,"?\0\n") erfragt werden, wobei n die
AES-ID des fraglichen Accessories angibt. Der Rückgabewert stellt dann
die menu_id (oder -1) dar.

'menu_tree' ist ein über die AES-ID indiziertes Feld und gibt die
Zeiger auf die Menübäume der Applikationen an. Für Applikationen, die
kein Menü angemeldet haben, wird NULL geliefert. Die Zeiger können
auch über
hi=menu_bar((OBJECT*)n, -2);
lo=menu_bar((OBJECT*)n, -3);
erfragt werden, wobei n die AES-ID des fraglichen Programms angibt.

'flags' ist ein über das Fensterhandle indiziertes Feld, wobei die
einzelnen Bits folgende Bedeutung haben:
Bit 0: gesetzt=Fenster ist geöffnet
Bit 1: gesetzt=Fenster ist ikonifiziert
Einträge für nicht vorhandene Fenster sind nicht definiert!


Sonstiges:
Alternative wind_get-Routine für PureC:

int wnd_get(int w_hnd, int func, int *p1, int *p2, int *p3, int *p4)
{

 AESPB	c;
 c.contrl=_GemParBlk.contrl;
 c.global=_GemParBlk.global;
 c.intin=_GemParBlk.intin;
 c.intout=_GemParBlk.intout;

 _GemParBlk.contrl[0]=104;
 _GemParBlk.contrl[1]=2;
 _GemParBlk.contrl[2]=5;
 _GemParBlk.contrl[3]=0;
 _GemParBlk.contrl[4]=0;

 _GemParBlk.intin[0]=w_hnd;
 _GemParBlk.intin[1]=func;

 _crystal(&c);

 *p1=_GemParBlk.intout[1];
 *p2=_GemParBlk.intout[2];
 *p3=_GemParBlk.intout[3];
 *p4=_GemParBlk.intout[4];

 return(_GemParBlk.intout[0]);
}
